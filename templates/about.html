About

I currently live and work in Ecuador and use public, inter-provincial buses for both work and recreation. In reading the daily news here, I became aware that there are a fairly large number of bus accidents in Ecuador each year. I wished there was a way to track these accidents and search out any patterns in the data so that one could utilize Ecuador's bus system in the safest possible manner - and so this project began.

Raw vs. Processed Data

The raw data table contains duplicates of certain accidents (whenever multiple news stories were published about an accident) and therefore does not make any attempts to aggregate death totals from multiple stories about the same accident. the raw data table also does not make any attempt to convert non integer accident times (i.e. "medianoche", "madrugada") into integer times.

The processed data table automates the process of combining then dropping duplicate rows and converts non integer times into integers so that the table can be read clearly and easily right off the bat and be immediately useful in analysis.

Tech Stack

This project was created in Python 3.6 using Flask as the web framework, BeautifulSoup for data retrieval and Pandas for data cleaning.


Methodology

1.
The pipeline begins with retrieving every news story about fatal bus accidents in Ecuador published in the Quito based El Commercio. This preliminary retrieval was done using the El Commercio website's basic search function to query "bus accidente ecuador."
2.
All of those stories are then retrieved and filtered for mentions of fatalities.
3.
The remaining stories about fatal accidents are then scanned for the exact death count, the accident time, the accident date and the accident route.
